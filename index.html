Here are some of my projects

  • On-Premises Migration #1

    IAM, MFA, Users, Groups, and Permissions.

    Planning is definitely key to ensuring a successful migration, especially when it comes to moving a large team. My role, as a Cloud Specialist was to create a migration solution in an automated way.

    In addition to migrating an IT team of 100 users, it's also important to associate permissions to the user groups. And enabling Multi-factor Authentication (MFA) for users adds an extra layer of security to this process.

  • On-Premises Migration #2

    EC2, VPC, RDS, Private and Public Subnets, Availability Zone (AZ).

    In this project, I was responsible for migrating a workload running in an Enterprise Data Center to AWS. My mission was to migrate the application and its database using Amazon EC2 and RDS services.

    In the solution architecture on AWS, VPC was implemented along with its Subnets, an EC2 instance to store the Application, and an RDS instance to store the Database.

    For this application to be accessible via the Internet, I have attached to my VPC an Internet Gateway implementation.

  • E-commerce Deployment.

    EC2, VPC, Magento, Ansible, Terraform, PHP, MySql, Redis, and, Availability Zone (AZ).

    MVP(Minimal Viable Product) Deployment of an e-commerce system in the AWS Cloud aims to validate whether the e-commerce store is a viable product for the market or not. Considering the limited time,the deployment will be done using automation tools such as Terraform to provision EC2 infrastructure and Ansible to automate the process of e-commerce provisioning by installing all software stacks required using its playbook.This solution will use Magento(An OpenStack application which runs the e-commerce website), PHP, MySql(database),and Redis(cache system which speeds up the user experience) as the stack of tools.

    Once the initail phase of this project gains some traction, the next phase of this project will be to deploy this architecture in a more robust way with high availability and the database running outside thesame virtual machine.
